real_parent 和 parent 通常是一样的，但是也会有另外的情况存在。例如，bash 创建一个进程，那进程的 parent 和 real_parent 就都是 bash。如果在 bash 上使用 GDB 来 debug 一个进程，这个时候 GDB 是 parent，bash 是这个进程的 real_parent。

每次调度时，从上到下遍历每个调度类，查找进程。相当于每个调度类维护了一个优先级队列，使用不同的算法进行排序。调度时，查找优先级最高的调度类中优先级最高的那个进程。

调度算法

CFS（完全公平调度算法）

系统维护每个进程占用CPU的时间，然后优先调度占用CPU时间少的进程，来保证各个进程之间的公平。

每个进程之间的优先级通过相同的绝对时间，针对不同进程进行不同的折算体现。例如，运行1ms，高优先级进程只算0.5ms，低优先级进程算2ms，那高优先级进程每运行4ms，低优先级进程才会运行1ms。

调度队列与调度实体

调度队列使用红黑树将所有进程的调度实体关联起来，快速在其中查找优先级最高的进程，并调整进程的位置。

主动调度

进程执行IO操作或类似操作时，调用schedule函数，主动让出CPU。此时会找调度队列中优先级最高的进程使用CPU。

抢占调度

抢占调度触发：

  1. 进程的时间片到
  2. 有高优先的进程插入

抢占式调度不会直接将当前进程挂起，而是标记该进程应该被抢占，然后在合适的实际进行调度。抢占时机如下：

  1. 从系统调用返回用户态
  2. 从中断返回用户态
  3. 内核态启动可抢占（有的内核进程在执行特殊任务时可以设置自身为不可抢占，当回复为可抢占时尝试进行抢占式调度）
  4. 从中断返回内核态
